## 1. 项目概览

BeerGaao 是一个全栈 Web 应用，提供股票技术分析、LLM 驱动的智能投资分析、策略回测等功能。

**技术栈:**

| 层级 | 技术 |
|------|------|
| 前端框架 | React 19 + TypeScript |
| 路由 | React Router v7 |
| 图表 | Lightweight Charts |
| 样式 | CSS Variables + Inline Styles |
| 构建工具 | Vite 6 + Tailwind CSS v4 |
| 后端 | Python FastAPI |
| 认证 | Supabase Auth |
| 数据源 | 东方财富 / 腾讯财经 / 新浪财经 |
| LLM | DeepSeek / 智谱 / SiliconFlow (可配置) |

## 4. 核心模块详解

### 4.1 数据层 (src/data/)

#### StockInfo 接口

```typescript
interface StockInfo {
  code: string;           // 股票代码
  name: string;           // 股票名称
  exchange: string;       // 交易所
  price: number;          // 当前价格
  change: number;         // 涨跌额
  changePercent: number;  // 涨跌幅
  prevClose: number;      // 昨收
  open: number;           // 开盘价
  high52: number;         // 52周最高
  low52: number;          // 52周最低
  marketCap: string;      // 市值
  pe: number;             // 市盈率
  dividendYield: string;  // 股息率
  eps: number;            // 每股收益
  roe: string;            // 净资产收益率
  grossMargin: string;    // 毛利率
}
```

#### 数据流

```
用户输入股票代码
       │
       ▼
fetchStockDetail(code)
       │
       ├──→ 腾讯 API (qt.gtimg.cn)  ← 主数据源
       │         │
       │         ▼
       │    解析 GBK 编码响应
       │    提取: 价格/PE/PB/ROE/市值
       │
       ├──→ fetchFundamentals(code)  ← 基本面补充
       │         │
       │         ▼
       │    东方财富 API
       │    获取: 股东/财务/行业
       │
       └──→ 返回 StockInfo 对象
```

#### 同业对比数据流 (Peers)

```
fetchPeers(code)
       │
       ▼
/api/em/peers?code=300475
       │
       ├── Tier 1: 东方财富动态行业查询
       │    - 获取股票行业名称 (f127)
       │    - 查询行业板块成分股
       │    - 返回 Top 10 同行 PE/PB/ROE
       │
       ├── Tier 2: 静态 sectorPeers 映射
       │    - 硬编码 30+ 热门板块同业
       │    - 腾讯批量行情 API 获取数据
       │
       └── Tier 3: 新浪 + 腾讯回退
            - 新浪行业板块搜索
            - 腾讯获取 PE/PB/ROE
```

### 4.2 LLM 分析引擎 (src/data/analysis.ts)

#### 分析模块

| 模块 | 函数 | 输出 |
|------|------|------|
| 交易信号 | `analyzeSignal()` | BUY/SELL/HOLD + 置信度 + AI论点 |
| 投资者观点 | `analyzeInvestors()` | 13位著名投资者的独立分析 |
| 估值分析 | `analyzeValuation()` | DCF/Owner Earnings/Damodaran 估值 |
| 情绪分析 | `analyzeSentiment()` | 正面/中性/负面比例 + 新闻标题 |
| 风险分析 | `analyzeRisk()` | 波动率/Beta/VaR/Taleb指标 |
| 综合分析 | `analyzeAll()` | 单次 LLM 调用完成全部5项分析 |
| 流式分析 | `analyzeAllStream()` | 流式输出 + 进度回调 |

#### LLM 调用流程

```
analyzeAll(stock, indicators)
       │
       ▼
buildCombinedPrompt(stock, indicatorStr)
       │  - 构建包含所有分析要求的 Prompt
       │  - 指定 JSON 输出格式
       │
       ▼
ask(SYSTEM_PROMPT, prompt)
       │
       ▼
llm.ts → POST /api/llm/chat
       │
       ▼
server/llm_client.py
       │  - 调用配置的 LLM API
       │  - 支持 DeepSeek/智谱/SiliconFlow
       │
       ▼
extractJSON(response)
       │  - 从 LLM 响应中提取 JSON
       │  - 处理 markdown 代码块
       │
       ▼
返回 AllAnalysisResult
```

## 11. 关键设计决策

### 11.1 为什么使用 Vite 代理而不是独立后端?

- **开发便利**: 前后端同域，无跨域问题
- **数据转换**: 在代理层统一处理不同数据源的格式差异
- **回退策略**: 便于实现多数据源回退逻辑

### 11.2 为什么使用 LLM 模拟投资者?

- **教育价值**: 展示不同投资风格的分析视角
- **多角度分析**: 避免单一模型偏见
- **用户友好**: 知名投资者名称更易理解

### 11.3 为什么使用 localStorage 缓存?

- **即时可用**: 无需后端缓存服务
- **12小时 TTL**: 平衡数据新鲜度和 API 调用成本
- **双层缓存**: Memory (快) + localStorage (持久)

### 11.4 为什么使用三层回退策略?

- **数据可靠性**: 单一 API 可能不可用
- **覆盖范围**: 不同 API 覆盖的股票范围不同
- **性能优化**: 静态映射最快，动态查询最准
