# Loop Engineering

## 一、什么是 Loop Engineering

由 OpenClaw 作者 Peter Steinberger 点火、Google Cloud AI 总监 Addy Osmani 正式定义：「Loop engineering 就是把『亲自给 agent 写 prompt 的那个你』替换掉，你转而去设计那个代替你做这件事的系统」。

过去与 coding agent 的协作是回合制——你写 prompt、读输出、再写下一句，全程握着 agent，一回合不能松手。Loop Engineering 要做的是循环制——你把「发现任务、布置任务、检查结果、决定下一步」设计成一个能自己运转的循环，让循环去握着 agent。

工作没有变容易，是杠杆的支点移动了：以前写一条好 prompt，收益是「这一次回答变好」；现在设计一个好 loop，收益是「之后每一次循环都变好」。

## 二、一个 loop 的六大核心部件

拆开真正跑起来的 loop，零件出奇地一致：

| 部件 | 职责 | 不装会怎样 |
|------|------|------------|
| 自动化 | loop 的心跳，定时或事件触发 | 每次手动启动，不算 loop，只是一个跑过一次的任务 |
| worktree | 给每个 agent 独立工作目录和分支，共享同一份仓库历史 | 多个 agent 同时改同一个文件，互相打架 |
| skill | 把项目规范、约定、坑写成文件，让 agent 该用时自己读 | 每个会话冷启动，像对金鱼一样把项目重新解释一遍 |
| connector | 基于 MCP 协议连接外部系统（工单、监控、消息、PR） | 只能看见文件系统，撑死了算半个 loop |
| sub-agent | 写代码的 agent 和检查代码的 agent 分开 | 写代码的模型给自己的作业打分时实在太手下留情 |
| 记忆（磁盘） | 把状态写在磁盘上，而非上下文里 | 模型在两次运行之间忘掉一切，今天的循环干了什么明天一概不知 |

「agent 会忘，但 repo 不会。」

## 三、一个真实的 loop 长什么样

以 Osmani 自己用的「晨间 triage loop」为例：

1. 每天早晨，自动化准时触发，调用分诊 skill
2. skill 读取昨天的 CI 失败记录、未关闭的 issue、最近提交，把「哪些问题值得处理」写进状态文件
3. 对每一个值得做的问题，开一个隔离的 worktree，派 sub-agent 进去起草修复
4. 第二个 sub-agent 对照 skill 规范和现有测试，把草稿审一遍
5. 审过了，connector 自动开 PR、更新对应工单
6. loop 搞不定的问题，丢进待办收件箱，等真人来看
7. 所有经过写回状态文件，明天从今天停下的地方继续

整个过程里，你只设计了一次，中间没有任何一步你写过 prompt

## 四、工具现状：现在就能搭

一年前搭一个 loop 意味着写一堆 bash 脚本；现在五大件全部内置在主流产品里（Codex / Claude Code）。

两家产品都有 /goal 类能力——不是「按节奏重复跑」，而是跑到你写的条件为真才停（如「所有测试通过且 lint 干净」），且每一轮结束后由独立的模型判断条件是否达成。

实操示例：在 Claude Code 里，用一条 /loop 命令就能把「PR 提交后等 CI、挂了就修、修完再推、全绿了通知我」整个流程交出去：

```
/loop 10m 检查当前分支 PR 的 CI 状态：有失败的检查就读日志、修复、推送；全部变绿后停下来，给我一句话总结改了什么
```

10m 是心跳（每 10 分钟自动醒来跑一轮），中间是任务，最后是停止条件加汇报。

部件是一件一件长出来的，不是一天配齐的。先让最小的循环转起来，觉得不放心再加检查的 sub-agent，想同时盯多个 PR 再上 worktree。

loop 的设计正在变得工具无关——部件是同样的部件，差的只是商标。值得积累的是图纸，不是对某家工具的肌肉记忆。

## 五、三盆冷水

## 1. 验证仍然归你
无人值守运行的 loop，也是无人值守犯错的 loop。检查 agent 嘴里的「done」只是一个声明，不是一个证明。

## 2. 理解债
loop 交付你没写过的代码越快，「仓库里实际存在的东西」和「你脑子里真正理解的东西」之间的鸿沟就越大。代码可能不烂，但你不知道它为什么是对的。

## 3. 认知投降
loop 自己转起来后，最舒服也最危险的姿势是：不再对产出有自己的观点，它给什么就收什么。

「带着判断力去设计 loop，它是解药；为了逃避思考去设计 loop，它是助燃剂。同一个动作，相反的结果。」

另外还有一笔现实账：token 成本——loop 是按循环烧 token 的，sub-agent 每多一个就多一份开销。

## 最终三句话

1. **四个词是一场瓶颈迁移**：prompt → context → harness → loop，模型越强瓶颈越往外移，最后移到「亲自按回车的你」身上
2. **一个 loop = 五大件 + 磁盘记忆**：自动化是心跳，worktree 防打架，skill 治金鱼记忆，connector 摸到真实世界，sub-agent 写查分离，状态文件让明天接得上今天
3. **loop 把杠杆变长了，但三笔账也变大了**：验证、理解债、认知投降——工具分不出你是在加速还是在逃避，你自己分得出
